# 我的 Markdown 研究笔记工作流

Markdown 对我最重要的价值并不是轻量，而是内容能够脱离某个应用长期存在。文本可以进入版本控制，也可以在不同工具之间自由迁移。

## 一个文件只承担一个目的

项目说明、每日记录、文献摘录和阶段总结分别维护。它们通过链接关联，而不是被塞进同一个不断增长的文档。

## 用元数据建立入口

每篇笔记至少包含日期、摘要、标签和相关项目。首页只展示摘要，正文保持完整的阅读空间。

## 让版本记录成为研究记录

提交信息会解释一次重要修改的原因。这样 Git 历史不仅保存文本差异，也保存项目判断的变化。

这套工作流并不追求复杂自动化。它只需要保证一件事：任何重要想法都能被找到，也能被重新放回它产生的语境中。
